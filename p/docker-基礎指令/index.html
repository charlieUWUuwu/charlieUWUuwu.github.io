# Start/Run a Container

  • docker pull <image_name>: pull an image from docker hub by default

  • doker run --name mymongo -d mongo: 執行一個新的 container (每次都會建置新的)

  • docker start mongo : 啟動名為 mongo 的 container

  • docker stop mongo: stop

  • --env(or -e): 傳遞環境參數到 container 中。例如運行mysql時常用MYSQL_RANDOM_ROOT_PASSWORD=yes

    • 可在第一次啟動時使用docker container logs指令到日誌中查找隨機密碼是什麼。
    • 在 docker hub 上會看到 mysql 有說密碼要指定三個選項的其中一個。
    • 官方 mysql image 已經移除 ps跟apt-get指令。可以改用 MariaDB

# What’s Going On In Conatainers

  • docker container top <container_ID>: list running processes in specific container
  • doker run --name mymongo -d mongo: 執行一個新的 container (每次都會建置新的)
  • docker container inspect : show metadata about the container(startup config, columes, networking, etc)
  • docker container stats : show live performance data for all containers

# Getting a Shell Inside Containers

  • docker container run -it: start new container interactively

  • docker container exec -it : run additional command in existing container

    • exec 是在現有的運行容器上運行一個附加的 process，所以不會影響 deamon program 的 root process
    • exec 適合用在故障排除, 稍微更改內容的情景，可以跳轉到 container 裡面

  • -t, --tty: pseudo-tty, similates a real terminal, like what SSH does

  • -i: interactive. Keep session open to receive terminal input

  • bash/sh: if run with -it, it will give you a terminal inside the running container

    • 用法 : docker container run -it --name proxy nginx bash
    • sh 更加輕巧，像是 Apline linux 只有 sh 而沒有 bash

# Others

  • --format: A common option for formatting the output of commands using “Go templates”
    • 例如docker container inspect --format "{{ .NetworkSettings.IPAddress }}" webhost，注意 Windows 的話要使用雙引號而非單引號
  • ps aux: show me all running processes
  • ps aux | grep mongo: 過濾出叫做 mongo 的 process
  • Apline Linux : 只有 5MB 的 Linux 的另一個發行版
