建立完整 React 專案

vite+Eslint+Prettier 建構 React 專案

Feb 22, 2025
3 minute read

以下紀錄使用 vite 建立專案，設置 Tailewind CSS，並搭配 Eslint + Prettier 作風格控制的步驟。

# 安裝

  • 去官網自行下載安裝
    • VS Code
    • Nodejs
  • VSCode 外掛安裝建議
    • ES7+ React/Redux/React-Native snippets
    • VSCode React Refactor
    • EditorConfig
    • ESLint
    • Prettier - Code formatter
    • Prettier ESLint

# 基本架構說明

當你使用 Vite 建立 React 專案時，預設的專案結構大致如下：

my-project/
├── node_modules/      # 安裝的 npm 套件
├── public/            # 公開資源（不經過 Vite 處理）
│   ├── favicon.svg    # 網站圖示
│   └── ...
├── src/               # 主要程式碼（React 相關）
│   ├── assets/        # 靜態資源（如圖片、CSS）
│   ├── components/    # 可重用的 React 元件
│   ├── pages/         # 分頁（通常使用 React Router）
│   ├── App.jsx        # 主應用程式組件
│   ├── main.jsx       # React 進入點
│   └── index.css      # 全域樣式
├── .gitignore         # Git 忽略的檔案
├── index.html         # 主 HTML 模板
├── package.json       # 專案資訊 & 依賴管理
├── vite.config.js     # Vite 設定檔
└── README.md          # 專案說明文件

# 詳細介紹

# public/

  • 放置不需要經 Vite 轉譯的靜態資源，例如 favicon.ico、圖片或外部 JS 檔案。
  • 這裡的文件會原封不動地被拷貝到 dist/，可直接用 /assets/image.png 來存取。

# src/

（主要開發目錄） (1) assets/

  • 存放靜態資源，如圖片、CSS、SVG 等，但會被 Vite 處理，可透過 import 使用： 
    import logo from '../assets/logo.png';

(2) components/

  • 存放可重複使用的 React 元件，例如： 
    // src/components/Button.jsx
const Button = ({ text }) => {
    return <button>{text}</button>;
};
export default Button;

(3) pages/（可選）

  • 若使用 React Router，通常會將每個頁面拆成獨立的 page 元件： 
    // src/pages/Home.jsx
const Home = () => <h1>Home Page</h1>;
export default Home;

(4) App.jsx

  • React 應用的主要組件，通常負責路由和全局狀態： 
    import { BrowserRouter, Route, Routes } from 'react-router-dom';
import Home from './pages/Home';
import About from './pages/About';

function App() {
    return (
        <BrowserRouter>
            <Routes>
                <Route path="/" element={<Home />} />
                <Route path="/about" element={<About />} />
            </Routes>
        </BrowserRouter>
    );
}

export default App;

(5) main.jsx

  • React 應用的入口點，將 App.jsx 掛載到 index.html： 
    import React from 'react';
import ReactDOM from 'react-dom/client';
import App from './App';
import './index.css';

ReactDOM.createRoot(document.getElementById('root')).render(
    <React.StrictMode>
        <App />
    </React.StrictMode>
);

# index.html

  • Vite 預設的 HTML 樣板，#root 是 React 掛載的地方： 
    <body>
    <div id="root"></div>
    <script type="module" src="/src/main.jsx"></script>
</body>

# vite.config.js

  • Vite 的設定檔，可以修改開發伺服器、路徑別名、環境變數等： 
    import { defineConfig } from 'vite';
import react from '@vitejs/plugin-react';

export default defineConfig({
    plugins: [react()],
    server: {
        port: 3000, // 指定開發伺服器埠號
    },
});

# 補充：專案進階架構

若是較大型的專案，通常會進一步拆分目錄：

my-project/
├── src/
│   ├── api/            # API 請求函式
│   ├── hooks/          # 自訂 Hooks
│   ├── context/        # React Context 狀態管理
│   ├── store/          # Redux 或 Zustand 狀態管理
│   ├── utils/          # 工具函式
│   ├── styles/         # SCSS / Tailwind / CSS

# 建立本地項目 

npm create vite@latest my-project1
cd my-project1
npm install
npm run dev           # 啟動開發預覽 server

安裝步驟直接看官網: https://tailwindcss.com/docs/installation/using-vite

# 基本格式規範（縮排、行尾符號等）

建立 .editorconfig 檔案，並輸入以下內容(來源為 fb)

# https://editorconfig.org
root = true

[*]
charset = utf-8
end_of_line = lf
indent_size = 2
indent_style = space
insert_final_newline = true
max_line_length = 80

[*.md]
max_line_length = 0

[COMMIT_EDITMSG]
max_line_length = 0

基礎設置到此就可以了，以下為進階選項

# 排版檢查 

npm i eslint            // 安裝
npm init @eslint/config // 設定eslint
  • eslint.config.js 設置成如下
import globals from 'globals';
import pluginJs from '@eslint/js';
import tseslint from 'typescript-eslint';
import pluginReact from 'eslint-plugin-react';
import reactRecommended from 'eslint-plugin-react/configs/recommended.js';

/** @type {import('eslint').Linter.Config[]} */
export default [
    // https://stackoverflow.com/questions/72780296/warning-react-version-not-specified-in-eslint-plugin-react-settings-while-run
    {
        files: ['**/*.{js,mjs,cjs,ts,jsx,tsx}'],
        ignores: ['dist/**/*'],
        ...reactRecommended,
        settings: {
            react: {
                version: 'detect',
            },
        },
    },
    pluginJs.configs.recommended,
    ...tseslint.configs.recommended,
    pluginReact.configs.flat.recommended,
    {
        languageOptions: {
            globals: {
                ...globals.browser,
                ...globals.es2021,
                ...globals.node,
            },
        },
        rules: {
            // https://coreui.io/blog/how-to-fix-react-must-be-in-scope-when-using-jsx/
            'react/react-in-jsx-scope': 'off', // 避免 {} 才視為一組
            'react/jsx-uses-react': 'off',
        }
    },
];
  • 在 package.json 中添加檢查指令
"scripts": {
    ...
    "lint": "eslint . --ext .vue,.js,.ts,.jsx,.tsx --fix",
    ...
    },
  • 接著執行以下指令就可以進行版面檢查
npm run lint

# 自動化排版 

npm i prettier -D
npm i eslint-config-prettier -D
npm i eslint-plugin-prettier -D
  • 在根目錄新增 .prettierrc，內容如下
{
  "printWidth": 100,
  "tabWidth": 4,
  "useTabs": false,
  "semi": true,
  "singleQuote": true,
  "quoteProps": "as-needed",
  "jsxSingleQuote": false,
  "trailingComma": "all",
  "bracketSpacing": true,
  "bracketSameLine": false,
  "arrowParens": "always",
  "requirePragma": false,
  "insertPragma": false,
  "proseWrap": "preserve",
  "htmlWhitespaceSensitivity": "css",
  "vueIndentScriptAndStyle": true,
  "endOfLine": "lf"
}
  • 在 eslint.config.js 中的 export default 陣列裡最後一個元素加入
// 開頭要導入
import prettier from 'eslint-config-prettier';
// 添加在 export default 的陣列裡
prettier
  • 在 package.json 中添加格式化設定
"format": "prettier --write \"./**/*.{html,vue,ts,tsx,js,json,md}\"",
  • 只要執行以下指令就可以自動排版
npm run format

# 存檔時自動排版

透過設定 VS Code setting，只要執行 ctrl+s時就觸發自動排版!

  • 打開 setting.json
    • 左下角齒輪 > Settings > 有全域的User跟專案範圍的Workspace可以選，選擇要的那個，往右上看有個文件上面有返回箭頭的符號，點擊!
  • 添加設置如下
 "editor.formatOnSave": true, //存檔時自動格式化
 "editor.defaultFormatter": "rvest.vs-code-prettier-eslint", // 解析規則
 "editor.formatOnSaveMode": "file", // 格式化範圍
